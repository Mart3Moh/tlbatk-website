# طلباتك — TLBATK | منصة الطلبات والتوصيل

موقع احترافي لخدمة **طلباتك** للأسر المنتجة والمتاجر: استلام، تخزين، تجهيز، شحن، تحصيل (الدفع عند الاستلام)، وتتبّع لحظي مع ربط منصّات المتاجر.
مبني بـ React + Vite + Tailwind + React Router، وجاهز للنشر على Vercel.

## الصفحات
- `/` — الصفحة الرئيسية (من نخدم، الخدمات، الربط مع المنصّات، الأسئلة).
- `/track` — صفحة تتبّع الطلب (يدخل العميل رقم الطلب ويشوف مراحله).

## ⚙️ التعديل السريع
كل إعداداتك في `src/config.ts`:
- `WHATSAPP_NUMBER` ← رقم الواتساب (مثال: `9665XXXXXXXX`)
- `BRAND` ← الاسم، الإيميل، المدينة، حسابات التواصل
- `INTEGRATIONS` ← المنصّات اللي تظهر في قسم الربط
- `TRACKING_API_URL` ← رابط الـ API للتتبّع (انظر تحت)

## 🔌 ربط التتبّع بـ API (سلة / زد / منصّة خاصة)
صفحة التتبّع تشتغل **بوضع تجريبي** بدون أي ربط. لتفعيل التتبّع الحقيقي:
1. جهّز عندك Endpoint يستقبل: `GET https://api.example.com/track?code=رقم_الطلب`
2. يرجّع JSON بنفس صيغة `TrackResult` الموجودة في `src/lib/tracking.ts`.
3. حط الرابط في `TRACKING_API_URL` داخل `src/config.ts`، أو الأفضل: أضف متغيّر بيئة في Vercel باسم `VITE_TRACKING_API_URL`.

> بمجرد ضبط الرابط، تتحوّل صفحة التتبّع تلقائياً للوضع الحقيقي بدون أي تعديل ثاني.

مثال شكل الـ JSON المتوقّع موجود معلّق داخل `src/lib/tracking.ts`.

## 💻 التشغيل محلياً
```bash
npm install
npm run dev
```

## 🏗️ البناء
```bash
npm run build
```

## 🚀 النشر على GitHub ثم Vercel
1. ارفع المشروع على مستودع GitHub.
2. vercel.com → New Project → اختر المستودع (يكتشفه Vite تلقائياً).
3. (اختياري) Settings → Environment Variables → أضف `VITE_TRACKING_API_URL`.
4. Deploy.

ملف `vercel.json` مضبوط مسبقاً ليعمل التوجيه (SPA) لصفحة `/track`.
